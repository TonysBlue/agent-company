# Design Context Policy

Version: company-context/v1-draft

The detailed development assurance design is pending Chairman review. Until approved, every task continues to use the currently implemented context system: shared constitution and public role map, executing-role private playbook, strategy/directive versions, repository/task contract, continuity, project history, handoffs, generation, fencing, and integrity checks.

After approval and implementation of the assurance kernel, C2/C3 task context will additionally include exact approved Goal, Design, Behavior Spec, Eval Contract, and Baseline references plus the artifact-set hash and gate decisions. These references are read-only and stale changes block execution. C0/C1 work receives a risk-scaled context and an audited rationale for the lighter process.

No role may treat the draft design as an active runtime rule, change task lifecycle behavior, or claim that a Goal/Design/Eval gate has passed until the corresponding implementation and durable evidence exist.
